# Camping List App 🏕️

A simple collaborative camping checklist for three families. It is a static web app (HTML/CSS/JavaScript) that stores shared data in Firebase Realtime Database.

## What the app does

- Shows camping items grouped by category.
- Lets each family mark which items they will bring.
- Tracks progress and per-family counts in real time.
- Lets users add, edit, and delete items.
- Syncs instantly for everyone using the same Firebase project.

## Project structure

- `index.html` – app layout and DOM containers.
- `styles.css` – all visual styling and responsive/mobile spacing.
- `firebase.js` – Firebase initialization and database helper methods.
- `app.js` – UI logic, rendering, event handlers, and modal flow.
- `netlify.toml` – static hosting config for Netlify.

## Firebase usage

The app uses **Firebase Realtime Database** with two top-level nodes:

- `items`
  - Each child key is auto-generated by `push()`.
  - Value structure:
    ```json
    {
      "name": "Sunscreen",
      "cat": "Health",
      "qty": "2 bottles",
      "note": "Buy before trip"
    }
    ```

- `checks`
  - Each child key is composite: `<itemId>_<familyIndex>`.
  - Value is boolean (`true` when checked).
  - Example: `-Nhx123_0: true`

### Real-time subscriptions

- `items` is subscribed with `onValue` to render categories and rows.
- `checks` is subscribed with `onValue` to render per-family check states.

### CRUD operations used

- Add item: `push(items, item)`
- Update item: `update(items/<itemId>, item)`
- Delete item: `remove(items/<itemId>)`
- Toggle check: `update(checks, { "<itemId>_<familyIndex>": true/false })`
- Cleanup checks on delete: set relevant check keys to `null` via `update`.

## Run locally

Because this app uses JavaScript modules (`type="module"`), run it with a local static server:

### Option A: Python

```bash
python3 -m http.server 8080
```

Then open: `http://localhost:8080`

### Option B: Node (if installed)

```bash
npx serve .
```

## Netlify deploy

This project is Netlify-ready with no build step.

1. Connect repository in Netlify.
2. Build command: *(leave empty)*
3. Publish directory: `.`
4. Deploy.

`netlify.toml` already sets:

- `publish = "."`
- SPA-style redirect to `index.html`.

## How to modify the app

- Add new UI behavior in `app.js`.
- Add/change styling in `styles.css`.
- Keep Firebase config and DB helpers in `firebase.js`.
- If adding new data fields, update:
  1. item form in `index.html`
  2. render logic in `app.js`
  3. write/update payloads in `firebase.js` call sites

## Notes

- This is intentionally framework-free for beginner-friendly maintenance.
- Firebase credentials are client-side config (normal for Firebase web apps).
